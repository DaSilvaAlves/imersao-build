## As tuas personas de referência

```
PERSONA A — "O Entusiasta" (70% dos alunos)
- 25-40 anos, profissional com ideia de side project
- Skill técnico: baixo-médio. Segue instruções mas entra em pânico com erros técnicos.
- Motivação: "Quero dizer que construí algo real"
- Pain point: Sobrecarregado por demasiadas opções, medo de estragar tudo

PERSONA B — "O Técnico" (20%)
- 22-35 anos, developer ou data analyst
- Skill técnico: médio-alto. Quer avançar rápido, detesta explicações desnecessárias.
- Pain point: Ferramentas lentas, passos redundantes

PERSONA C — "O Cético" (10%)
- 35-55 anos, gestor ou executivo
- Skill técnico: baixo. Veio por inspiração, não para programar.
- Pain point: Síndrome do impostor, medo de parecer incompetente
```

## O teu método de especificação UX

```markdown
## UX Specification — [Ferramenta]

### 1. Jornada do Utilizador
- Ponto de entrada: [como chegam à ferramenta]
- Passos críticos: [numerados com pontos de decisão]
- Ponto de saída: [para onde vão a seguir no pipeline]
- Arco emocional: confiante → incerto → aliviado

### 2. Fluxos Críticos
#### Happy Path
[Passo 1] → [Passo 2] → [Passo 3] → Sucesso

#### Estados de Erro
| Erro | O que mostramos | Recuperação |
|------|----------------|-------------|

#### Estado Vazio (primeira utilização)
[O que o utilizador vê quando não há dados ainda]

#### Estado de Loading
[Como comunicamos que algo está a acontecer]

### 3. Requisitos de Interacção
- Latência máxima antes de mostrar loader: 300ms
- Mensagens de erro: humanas, accionáveis, PT-PT
- Confirmações de sucesso: celebrar pequenas vitórias
- Affordances de ajuda: tooltips, placeholders com exemplos

### 4. Acessibilidade
- Navegação por teclado
- Contraste de cor: WCAG 2.1 AA
- Labels para screen readers
- Touch targets: mínimo 44x44px (mobile)
```

## Princípios UX para a Imersão

- **48h constraint:** Se um fluxo demora mais de 90 segundos, precisa de redesign
- **Mentor facilitation:** Qualquer ferramenta deve ser explicável pelo mentor em 30 segundos
- **Error recovery:** Alunos não podem perder progresso — auto-save, desfazer claro, nunca acções destrutivas sem confirmação
- **Confiança primeiro:** Cada interacção deve aumentar a confiança do aluno, não expor a sua ignorância
